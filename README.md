# WhatsApp AI-консультант для магазина штор

## Что это

Бот, который не просто отвечает по кнопкам, а ведёт живой диалог, отвечает
только на основе реального каталога (RAG — Retrieval-Augmented Generation),
может сам оформить заказ и передать диалог менеджеру, если нужно.

## Архитектура

```
Клиент → WhatsApp → Green API → POST /webhook (Express)
                                      ↓
                              Очередь (Redis + BullMQ)
                                      ↓
                    Поиск по каталогу (pgvector) + история диалога
                                      ↓
                          Claude API (генерация ответа + вызов инструментов)
                                      ↓
                    create_order / escalate_to_manager (реальные действия)
                                      ↓
                              Ответ клиенту через Green API
```

### 2. Заведите аккаунты

1. **Green API** (green-api.com) — регистрируетесь, создаёте инстанс,
   привязываете свой WhatsApp (сканируете QR в личном кабинете). Получаете
   `idInstance` и `apiTokenInstance`.
2. **Anthropic Console** (console.anthropic.com) — получаете `ANTHROPIC_API_KEY`.
3. **Voyage AI** (voyageai.com) — получаете `VOYAGE_API_KEY` для эмбеддингов каталога.

### 3. Настройте окружение

```bash
cp .env.example .env
# впишите туда все ключи из шага 2, а также DATABASE_URL и REDIS_URL
```

### 4. Создайте базу данных и таблицы

```bash
createdb curtains_bot
psql curtains_bot -f src/schema.sql
```

### 5. Установите пакеты и загрузите каталог

```bash
npm install
npm run seed   # загружает тестовые товары из src/seedCatalog.js
```

Замените товары в `src/seedCatalog.js` на свои реальные (или напишите
отдельный импорт из CSV/1С — структура та же: название, цена, размеры, описание).

### 6. Настройте вебхук в Green API

В личном кабинете Green API укажите URL вебхука:
```
https://ваш-домен.com/webhook
```
и заголовок `X-Webhook-Secret` со значением из вашего `.env` (WEBHOOK_SECRET).

Для локальной разработки используйте ngrok:
```bash
ngrok http 3000
```
и укажите полученный https-адрес как URL вебхука.

### 7. Запустите сервер

```bash
npm start
```

Проверка, что всё живо:
```bash
curl http://localhost:3000/health
```

### 8. Тестируйте

Напишите с любого телефона на номер, привязанный к Green API. Бот должен
ответить, используя каталог из базы, а не выдуманные данные.

## Важно для продакшена

- **Не тестируйте на основном номере магазина** — заведите отдельный тестовый
  WhatsApp-номер, чтобы не словить блокировку от Meta во время отладки.
- **Мониторинг** — добавьте алерты (например, в Telegram) на событие `failed`
  в очереди (`src/queue.js`, `worker.on('failed', ...)`), чтобы узнавать о
  сбоях сразу, а не через жалобы клиентов.
- **Бэкапы базы** — в таблице `orders` и `messages` — ваши реальные заказы и
  переписка, настройте регулярный бэкап PostgreSQL.
- **Расширение каталога** — при добавлении новых товаров не забывайте
  генерировать для них эмбеддинг (см. `seedCatalog.js`), иначе поиск их не найдёт.
- **Лимиты** — `queue.js` ограничивает 20 сообщений/сек в обработке, при росте
  нагрузки увеличивайте `concurrency`, но следите за лимитами Anthropic и Green API.

## Структура файлов

```
src/
  server.js          — Express, приём вебхуков
  queue.js            — очередь сообщений (надёжность, идемпотентность)
  claudeService.js     — промпт, вызов Claude, обработка инструментов (заказ/эскалация)
  catalogService.js    — эмбеддинги и семантический поиск по каталогу
  whatsappService.js   — отправка сообщений через Green API
  db.js               — подключение к PostgreSQL
  schema.sql          — структура базы данных
  seedCatalog.js       — загрузка товаров с генерацией эмбеддингов
```
